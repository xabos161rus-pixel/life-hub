# LifeHearth

Личный центр управления жизнью: задачи, цели, привычки, заметки, обучение, финансы, семья. PWA для iPhone/Mac — устанавливается на экран «Домой» из Safari, работает офлайн, при желании синхронизируется между устройствами с E2E-шифрованием.

## Архитектура

- **React 19 + TypeScript + Vite 8**, Tailwind CSS 4 (CSS-first, токены в `src/index.css`)
- **Dexie.js (IndexedDB)** — данные хранятся локально на устройстве; `useLiveQuery` как слой состояния
- **PWA** через vite-plugin-pwa: precache всего билда, `autoUpdate` (обновление подтягивается молча при перезапуске)
- **Бэкенд**: один Cloudflare Worker (`worker/`) — E2E-синхронизация (D1), push-напоминания (Web Push + cron) и семейный чат (Durable Object). Подробнее ниже.
- Хостинг фронта: GitHub Pages (`.github/workflows/deploy.yml`), base path `/life-hub/`

### Слой данных

Каждая запись наследует `BaseEntity` (`id` uuid, `createdAt`, `updatedAt`, `deletedAt` — мягкое удаление). Все записи идут **только** через `src/db/repo.ts` (`create`/`update`/`remove`), который проставляет sync-штампы и дёргает `scheduleSyncSoon()` — пачка правок уходит в облако одним синком.

Календарные даты — локальные строки `YYYY-MM-DD` (`src/lib/dates.ts`), чтобы отметка в 23:30 МСК не уезжала на другой день.

### Облачная синхронизация (E2E)

Движок `src/lib/sync.ts` + `SyncRunner.tsx`. Опциональна: пока не настроена, `runSync` сразу выходит и накладных нет.

- **End-to-end шифрование** (`src/lib/crypto.ts`): содержимое записей шифруется на устройстве; на Worker уходит только шифротекст + служебные поля (`id`, `updatedAt`, `deletedAt`). Сервер открытых данных не видит.
- **Сопряжение устройств**: первое устройство создаёт аккаунт (`createSyncAccount`), остальные подключаются по коду/QR (`connectSync`) — код несёт `accountId`, `authToken` и ключ шифрования. UI в Настройках (`features/settings/sync/`).
- **Цикл**: `pull` (получить чужие изменения, расшифровать, применить) → `push` (зашифровать свои свежие, отправить). Запускается при старте, возврате в приложение, раз в 60 c и через ~1.5 c после любой правки (debounce).
- **Конфликты**: last-write-wins по `updatedAt`. Удалённые правки пишутся в Dexie напрямую (минуя repo), чтобы сохранить серверный `updatedAt` и не зациклить синк.
- **Что синхронизируется**: задачи, проекты, цели, заметки, обучение, расходы, энергия, места, напоминания, метрики. Настройки устройства и секреты синхронизации — НЕ синхронизируются (намеренно).

### Push-напоминания

`src/lib/push.ts` + эндпоинты Worker'а (`/schedule`, `/cancel`, `/push-register`). Web Push (VAPID); iOS 16.4+ поддерживает push только для **установленных** PWA (standalone). Worker хранит подписку и по cron шлёт напоминание о задаче в назначенное время, даже когда приложение закрыто. Эндпоинт `/notify-update` — пинг всем устройствам о выходе новой версии.

### Семья (чат)

Реалтайм-чат через Durable Object `FamilyRoom` (`worker/src/familyRoom.js`), эндпоинты `/family/*`. Текст, фото и голосовые сообщения, ответы-цитаты (тап — прыжок к оригиналу), реакции (append-only записи, LWW по участнику), правка («изменено») и удаление своих сообщений, галочки доставки/прочтения, «печатает…» (эфемерный WS-броадкаст), разделители дат, приглашение по QR, пуш о новых сообщениях и общих задачах. Содержимое (включая фото и голос) E2E-шифруется общим ключом семьи — на Worker идёт только шифротекст.

### Аудиозвонки (WebRTC, 1:1)

Звонок между двумя участниками семьи (`src/lib/family/familyCall.ts`). Медиа идёт P2P между устройствами — серверу платить не за что. Сигналинг (SDP/ICE) проходит через тот же семейный WebSocket и E2E-шифруется семейным ключом (эфемерно, в БД не пишется). ICE-серверы отдаёт Worker (`/family/turn`): STUN бесплатен и работает из коробки; TURN-relay (для сетей за симметричным NAT) — Cloudflare Realtime free tier (1000 ГБ/мес), включается секретами `TURN_KEY_ID` / `TURN_KEY_API_TOKEN` (см. `worker/wrangler.toml`). Входящий ловится на любом экране (`CallRunner` в `App.tsx`); если адресат офлайн — пуш-нудж «Входящий звонок», а звонящий переотправляет offer, пока тот не подключится (в пределах 30 c). Ограничение iOS: нет «звонка поверх локскрина» (CallKit недоступен PWA) — входящий приходит как уведомление, ответ после открытия приложения.

### Бэкап

Помимо облачного синка есть локальный экспорт/импорт JSON (в Настройках) — данные целиком из IndexedDB. Бейдж на табе «Ещё» напоминает, если бэкапа не было больше 7 дней.

### Обучение нового пользователя

Два уровня: **вводный тур** (`features/onboarding/OnboardingOverlay.tsx`) — слайды по разделам при первом запуске, пока в настройках нет `onboardingDone`; и **контекстные подсказки** (`components/ui/Hint.tsx` + `hooks/useHint.ts`) — одноразовые советы в момент первого использования раздела (умный ввод дат, автонумерация по Enter, жесты, крестик очистки поля). Закрытая подсказка запоминается в `settings.seenHints` и больше не показывается; сброс всего обучения — в Настройках («Показать обучение заново»).

## Разработка

```bash
npm install
npm run dev      # http://localhost:5173/life-hub/
npm run build    # tsc + vite build + service worker
```

Бэкенд:

```bash
cd worker
npm install
npx wrangler dev      # локальный Worker (D1 + Durable Objects)
npx wrangler deploy   # деплой воркера
```

## Деплой

Push в `main` → GitHub Actions собирает фронт и публикует на GitHub Pages (~2 мин). На устройстве: перезапустить приложение — `autoUpdate` подтянет новую версию молча (на iOS иногда нужен второй запуск). Worker деплоится отдельно через `wrangler`.
